Programación Orientada a Objetos

Un curso introductorio sobre los fundamentos de la programación orientada a objetos y sus aplicaciones prácticas.

Primer Parcial

Inteligencia Artificial

La inteligencia artificial (IA) es una disciplina que busca desarrollar sistemas capaces de imitar la inteligencia humana, como el aprendizaje, la resolución de problemas y la toma de decisiones. Este tema cubre los fundamentos de IA, incluyendo:

  • Aprendizaje automático (Machine Learning): Algoritmos que permiten a las máquinas aprender de datos, como regresión lineal, árboles de decisión y redes neuronales.
  • Redes neuronales: Modelos computacionales inspirados en el cerebro humano, utilizados en aplicaciones como el reconocimiento de imágenes y el procesamiento del lenguaje natural.
  • Algoritmos de búsqueda: Como A* o búsqueda en profundidad, que se utilizan en sistemas expertos y juegos.

Las aplicaciones de la IA son vastas: en medicina, se utilizan para diagnosticar enfermedades; en automoción, para vehículos autónomos; y en entretenimiento, para sistemas de recomendación como los de Netflix o Spotify. Por ejemplo, los algoritmos de recomendación analizan patrones de consumo para sugerir contenido personalizado.

Ejemplo de Algoritmo: Regresión Lineal

La regresión lineal es un algoritmo de aprendizaje supervisado que modela la relación entre una variable dependiente (por ejemplo, el precio de una casa) y una o más variables independientes (como el tamaño o la ubicación). Un ejemplo práctico:

# Pseudocódigo para regresión lineal
Entradas: tamaño_casa (m²), precio_casa (USD)
Modelo: precio = a * tamaño + b
Entrenar modelo con datos históricos
Predecir precio para un tamaño dado

En Python, se implementaría usando bibliotecas como scikit-learn:

from sklearn.linear_model import LinearRegression
modelo = LinearRegression()
modelo.fit(tamaño_casa, precio_casa)
precio_predicho = modelo.predict(nuevo_tamaño)

Modelo Ágil

El desarrollo ágil es un enfoque iterativo e incremental para el desarrollo de software, que prioriza la flexibilidad, la colaboración y la entrega continua de valor al cliente. Las metodologías más comunes son:

  • Scrum: Organiza el trabajo en sprints (iteraciones de 1-4 semanas) con roles definidos (Product Owner, Scrum Master, equipo de desarrollo) y ceremonias como reuniones diarias (Daily Scrum) y revisiones de sprint.
  • Kanban: Utiliza tableros visuales para gestionar el flujo de trabajo, limitando el trabajo en curso para mejorar la eficiencia.

Las ventajas incluyen una mayor adaptabilidad a los cambios, una mejor comunicación en el equipo y entregas más rápidas. Por ejemplo, en un proyecto de desarrollo de una aplicación móvil, el equipo puede ajustar las funcionalidades según retroalimentación del usuario tras cada sprint.

Ejemplo de Tablero Kanban

Un tablero Kanban típico tiene columnas como:

  • Por hacer: Tareas pendientes.
  • En progreso: Tareas en curso.
  • Completado: Tareas terminadas.

Herramientas como Trello o Jira facilitan la implementación de Kanban en equipos remotos.

Segundo Parcial

Modelo Entidad-Relación

El modelo entidad-relación (ER) es una herramienta para diseñar bases de datos relacionales, representando gráficamente las entidades, sus atributos y las relaciones entre ellas. Es fundamental para estructurar datos de manera lógica y eficiente.

  • Entidades: Objetos del mundo real (por ejemplo, "Libro", "Autor").
  • Atributos: Propiedades de las entidades (por ejemplo, "Título" o "ID").
  • Relaciones: Conexiones entre entidades (por ejemplo, "Autor escribe Libro").

Un diagrama ER ayuda a visualizar la estructura de una base de datos antes de implementarla. Por ejemplo, en un sistema de gestión de biblioteca, las entidades principales serían "Libro", "Autor" y "Usuario", con relaciones como "escrito por" o "prestado a".

Ejemplo Avanzado de Diagrama ER

Consideremos un sistema de biblioteca más complejo:

  • Libro: (ID_Libro, Título, ISBN, Año_Publicación, AutorID)
  • Autor: (ID_Autor, Nombre, Nacionalidad)
  • Usuario: (ID_Usuario, Nombre, Email, Teléfono)
  • Préstamo: (ID_Préstamo, ID_Libro, ID_Usuario, Fecha_Préstamo, Fecha_Devolución)

Relaciones: Un autor puede escribir varios libros (1:N), un libro puede ser prestado a varios usuarios a lo largo del tiempo (1:N), y un usuario puede tener múltiples préstamos (1:N).

Un diagrama ER se puede crear usando herramientas como MySQL Workbench o Lucidchart.

Bases de Datos

Las bases de datos relacionales organizan datos en tablas relacionadas mediante claves primarias y foráneas. SQL (Structured Query Language) es el estándar para interactuar con estas bases de datos, permitiendo operaciones como:

  • CRUD: Crear (INSERT), Leer (SELECT), Actualizar (UPDATE), Eliminar (DELETE).
  • Normalización: Proceso para eliminar redundancias y garantizar la integridad de los datos.
  • Consultas avanzadas: Uso de JOIN, GROUP BY y subconsultas.

Por ejemplo, en una base de datos de biblioteca, se puede usar SQL para consultar los libros prestados por un usuario específico.

Ejemplo de Consulta SQL Compleja

Para obtener los libros prestados por un usuario, incluyendo el nombre del autor:

SELECT L.Título, A.Nombre, P.Fecha_Préstamo
FROM Libros L
JOIN Préstamos P ON L.ID_Libro = P.ID_Libro
JOIN Autores A ON L.AutorID = A.ID_Autor
WHERE P.ID_Usuario = 1;

Esta consulta combina tres tablas para obtener información detallada, demostrando el poder de las uniones (JOIN) en SQL.

Tercer Parcial

Java

Java es un lenguaje de programación orientado a objetos, robusto, portable y ampliamente utilizado en aplicaciones empresariales, móviles (Android) y web. Los conceptos clave de OOP en Java incluyen:

  • Clases y objetos: Una clase es una plantilla; un objeto es una instancia de esa plantilla.
  • Herencia: Permite que una clase herede atributos y métodos de otra.
  • Polimorfismo: Permite que un mismo método se comporte de manera diferente según el objeto.
  • Encapsulación: Oculta los detalles internos de un objeto, exponiendo solo lo necesario.

Java es ideal para proyectos grandes debido a su tipado fuerte y su gestión automática de memoria (Garbage Collection).

Ejemplo de Herencia en Java

public class Persona {
    protected String nombre;
    protected int edad;

    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    public void mostrarInfo() {
        System.out.println("Nombre: " + nombre + ", Edad: " + edad);
    }
}

public class Estudiante extends Persona {
    private String matricula;

    public Estudiante(String nombre, int edad, String matricula) {
        super(nombre, edad);
        this.matricula = matricula;
    }

    @Override
    public void mostrarInfo() {
        super.mostrarInfo();
        System.out.println("Matrícula: " + matricula);
    }
}

public class Main {
    public static void main(String[] args) {
        Estudiante estudiante = new Estudiante("Ana", 20, "12345");
        estudiante.mostrarInfo();
    }
}

Este ejemplo muestra herencia y polimorfismo, donde la clase Estudiante extiende Persona y sobrescribe el método mostrarInfo.

Visual Basic

Visual Basic (VB) es un lenguaje orientado a eventos, ideal para desarrollar aplicaciones de escritorio con interfaces gráficas (GUI). Utiliza un entorno de desarrollo integrado como Visual Studio, donde los programadores pueden diseñar formularios y asociar eventos a controles.

  • Controles: Botones, etiquetas, cuadros de texto, etc.
  • Eventos: Acciones del usuario, como clics o cambios en un cuadro de texto.
  • Propiedades: Configuraciones de los controles, como el texto o el color.

VB es ideal para prototipos rápidos y aplicaciones empresariales simples, como sistemas de inventario o formularios de entrada de datos.

Ejemplo de Aplicación en Visual Basic

Un formulario con un cuadro de texto y un botón para saludar al usuario:

Public Class Form1
Private Sub Button1_Click(sender As Object, e As EventArgs) Handles Button1.Click
Dim nombre As String = TextBox1.Text
MessageBox.Show("¡Hola, " & nombre & "!")
End Sub
End Class

En este ejemplo, el evento Button1_Click se dispara al hacer clic en el botón, mostrando un mensaje personalizado basado en el texto ingresado.
