Programación Orientada a Objetos

Un curso introductorio sobre los fundamentos de la programación orientada a objetos y sus aplicaciones prácticas.

Primer Parcial

Modelo Espiral

El modelo espiral es un enfoque de desarrollo de software que combina elementos del modelo en cascada y el desarrollo iterativo. Es particularmente útil para proyectos grandes y complejos donde los riesgos deben gestionarse cuidadosamente. Cada iteración (o espiral) incluye las siguientes fases:

  • Determinación de objetivos: Identificar metas, restricciones y alternativas.
  • Análisis de riesgos: Evaluar riesgos técnicos, financieros y operativos.
  • Desarrollo y prueba: Construir prototipos o versiones iniciales del software.
  • Revisión y planificación: Evaluar resultados y planificar la siguiente iteración.

Este modelo es ideal para sistemas críticos, como software médico o aeroespacial, donde la mitigación de riesgos es crucial. Por ejemplo, en el desarrollo de un sistema de control de tráfico aéreo, el modelo espiral permite probar prototipos en cada iteración para garantizar seguridad y funcionalidad.

Ejemplo de Aplicación del Modelo Espiral

Imagina un proyecto para desarrollar una aplicación de banca en línea:

  • Iteración 1: Diseñar un prototipo de autenticación de usuarios, evaluando riesgos de seguridad.
  • Iteración 2: Implementar transferencias bancarias, probando la integridad de las transacciones.
  • Iteración 3: Agregar funciones de notificación, ajustando según retroalimentación.

Cada iteración reduce riesgos y mejora el producto, culminando en una versión final robusta.

Modelo Entidad-Relación

El modelo entidad-relación (ER) es una herramienta para diseñar bases de datos relacionales, representando gráficamente las entidades, sus atributos y las relaciones entre ellas. Es fundamental para estructurar datos de manera lógica y eficiente.

  • Entidades: Objetos del mundo real (por ejemplo, "Estudiante", "Curso").
  • Atributos: Propiedades de las entidades (por ejemplo, "Nombre" o "ID_Estudiante").
  • Relaciones: Conexiones entre entidades (por ejemplo, "Estudiante se inscribe en Curso").

Un diagrama ER visualiza la estructura de una base de datos antes de su implementación. Por ejemplo, en un sistema de gestión escolar, las entidades principales serían "Estudiante", "Curso" y "Profesor", con relaciones como "impartido por" o "inscrito en".

Ejemplo de Diagrama ER

Un sistema escolar simple podría incluir:

  • Estudiante: (ID_Estudiante, Nombre, Email)
  • Curso: (ID_Curso, Nombre, Créditos)
  • Profesor: (ID_Profesor, Nombre, Departamento)

Relaciones: Un profesor imparte varios cursos (1:N), y un estudiante se inscribe en varios cursos (M:N). Herramientas como Lucidchart o MySQL Workbench facilitan la creación de diagramas ER.

Segundo Parcial

Entidad-Relación Avanzado

El modelo entidad-relación avanzado extiende los conceptos básicos con elementos como entidades débiles, claves foráneas, y relaciones más complejas. Se enfoca en modelar sistemas con dependencias y restricciones específicas.

  • Entidades débiles: Entidades que no tienen clave primaria propia y dependen de otra entidad (por ejemplo, "Préstamo" depende de "Libro").
  • Claves foráneas: Atributos que vinculan tablas en una base de datos relacional.
  • Relaciones ternarias: Conexiones que involucran tres entidades, como un estudiante inscrito en un curso con un profesor específico.

Por ejemplo, en un sistema de biblioteca, una entidad débil "Préstamo" podría depender de "Libro" y "Usuario", con atributos como fecha de préstamo y devolución.

Ejemplo Avanzado de Diagrama ER

Un sistema de biblioteca avanzado incluye:

  • Libro: (ID_Libro, Título, ISBN, Año_Publicación)
  • Autor: (ID_Autor, Nombre, Nacionalidad)
  • Usuario: (ID_Usuario, Nombre, Email, Teléfono)
  • Préstamo: (ID_Libro, ID_Usuario, Fecha_Préstamo, Fecha_Devolución)

Relaciones: Un autor escribe varios libros (1:N), un libro puede ser prestado a varios usuarios (1:N), y "Préstamo" es una entidad débil que depende de "Libro" y "Usuario".

Cardinalidad

La cardinalidad define el número de instancias de una entidad que pueden relacionarse con otra en un modelo ER. Es esencial para diseñar bases de datos precisas y garantizar la integridad de los datos.

  • Uno a uno (1:1): Una instancia de una entidad se relaciona con exactamente una instancia de otra (por ejemplo, un estudiante tiene un único número de identificación).
  • Uno a muchos (1:N): Una instancia de una entidad se relaciona con varias instancias de otra (por ejemplo, un autor escribe varios libros).
  • Muchos a muchos (M:N): Varias instancias de una entidad se relacionan con varias instancias de otra (por ejemplo, estudiantes inscritos en múltiples cursos).

La cardinalidad se representa en diagramas ER con notaciones como líneas, flechas o números. Por ejemplo, en un sistema escolar, la relación "Estudiante-Curso" es M:N, requiriendo una tabla intermedia para gestionar las inscripciones.

Ejemplo de Cardinalidad en SQL

Para modelar una relación M:N entre "Estudiante" y "Curso":

CREATE TABLE Estudiante (
    ID_Estudiante INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Email VARCHAR(100)
);

CREATE TABLE Curso (
    ID_Curso INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Créditos INT
);

CREATE TABLE Inscripcion (
    ID_Estudiante INT,
    ID_Curso INT,
    Fecha_Inscripcion DATE,
    PRIMARY KEY (ID_Estudiante, ID_Curso),
    FOREIGN KEY (ID_Estudiante) REFERENCES Estudiante(ID_Estudiante),
    FOREIGN KEY (ID_Curso) REFERENCES Curso(ID_Curso)
);

La tabla Inscripcion maneja la relación M:N, permitiendo que un estudiante se inscriba en múltiples cursos y un curso tenga múltiples estudiantes.

Tercer Parcial

Conexión a Java con MySQL

Java permite conectar aplicaciones con bases de datos MySQL utilizando JDBC (Java Database Connectivity), una API que facilita la ejecución de consultas SQL desde código Java. Esto es esencial para aplicaciones dinámicas que requieren almacenamiento y recuperación de datos.

  • Driver JDBC: Biblioteca que actúa como puente entre Java y MySQL (por ejemplo, Connector/J).
  • Conexión: Establecer una conexión con la base de datos usando una URL, usuario y contraseña.
  • Consultas: Ejecutar operaciones CRUD mediante sentencias SQL.

Por ejemplo, una aplicación de gestión de biblioteca puede usar Java para consultar libros disponibles o registrar préstamos en una base de datos MySQL.

Ejemplo de Conexión Java-MySQL

Este código conecta a una base de datos MySQL y recupera todos los libros:

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.Statement;

public class ConexionMySQL {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/biblioteca";
        String usuario = "root";
        String contraseña = "password";

        try {
            // Cargar el driver JDBC
            Class.forName("com.mysql.cj.jdbc.Driver");

            // Establecer conexión
            Connection conexion = DriverManager.getConnection(url, usuario, contraseña);

            // Crear una consulta
            Statement stmt = conexion.createStatement();
            ResultSet rs = stmt.executeQuery("SELECT Título, Año_Publicación FROM Libros");

            // Procesar resultados
            while (rs.next()) {
                System.out.println("Título: " + rs.getString("Título") + ", Año: " + rs.getInt("Año_Publicación"));
            }

            // Cerrar conexión
            rs.close();
            stmt.close();
            conexion.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Este ejemplo asume una base de datos "biblioteca" con una tabla "Libros". Requiere el driver MySQL Connector/J en el proyecto.

Uso de Base de Datos

El uso de bases de datos en aplicaciones implica diseñar, consultar y gestionar datos de manera eficiente. En el contexto de OOP, las bases de datos relacionales como MySQL se integran con objetos Java para modelar datos del mundo real.

  • CRUD: Crear, leer, actualizar y eliminar registros.
  • Normalización: Organizar datos para evitar redundancias.
  • ORM (Object-Relational Mapping): Herramientas como Hibernate mapean objetos Java a tablas de bases de datos.

Por ejemplo, en una aplicación de gestión de inventario, se pueden usar consultas SQL para rastrear productos y clases Java para modelar objetos como "Producto" o "Categoría".

Ejemplo de Operación CRUD en Java

Insertar un nuevo libro en la base de datos:

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;

public class InsertarLibro {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/biblioteca";
        String usuario = "root";
        String contraseña = "password";

        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
            Connection conexion = DriverManager.getConnection(url, usuario, contraseña);

            String sql = "INSERT INTO Libros (Título, ISBN, Año_Publicación) VALUES (?, ?, ?)";
            PreparedStatement stmt = conexion.prepareStatement(sql);
            stmt.setString(1, "Cien Años de Soledad");
            stmt.setString(2, "978-3-16-148410-0");
            stmt.setInt(3, 1967);

            int filasAfectadas = stmt.executeUpdate();
            System.out.println("Filas insertadas: " + filasAfectadas);

            stmt.close();
            conexion.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Este ejemplo usa PreparedStatement para insertar datos de forma segura, evitando inyecciones SQL.
